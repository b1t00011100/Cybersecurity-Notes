# Tree

## Definition

<aside>

*The topmost node of the tree is called the root, and the nodes below it are called the child nodes. Each node can have multiple child nodes, and these child nodes can also have their own child nodes, forming a recursive structure*

## Terminology of Tree Data Structure

- **Parent Node**- Predecessor of root node.
- **Child Node**- immediate successor of that Node.
- **Root Node**- Topmost Node.
- **Siblings**- Children of the same parent.
- **Level of a Node** - The count of edges on the path from the root node to that node, e.g Root node is at 0 level
- **Neighbour of a node**- Parent or child nodes of that node are called neighbors of that node.

## Properties of Tree Data Structure

- **Number of edges**- An edge can be defined as connection between nodes.
- **Depth of a Node**- Distance/path from root to that node is the depth of that node
- **Height of a node:** The height of a node can be defined as the length of the longest path from the node to a leaf node of the tree.
- **Height of the Tree:** The height of a tree is the length of the longest path from the root of the tree to a leaf node of the tree.
- **Degree of a Node-** The total count of sub-trees attached to that node is the degree of that node**.**

## Traversal of Binary Tree

**The Traversal of a Binary tree is to visit each node exactly once**.

There are three Orders in which this executes:

1. Pre-Order : ROOT - LEFT SUB-TREE - RIGHT SUB-TREE
2. In-Order : LEFT SUBTREE - ROOT - RIGHT SUB-TREE
3. Post-Order : LEFT SUB-TREE - RIGHT SUB-TREE - ROOT
